\section{Projeto do sistema\index{Projeto do sistema}\label{sec:Projeto-do-sistema}}

O software desenvolvido foi implementado com a linguagem C++, sob
o paradigma de orientação ao objeto. 

Inicialmente, foi utilizado a biblioteca \textit{SFML }para a criação
de janelas para o usuário, e utilizado o ambiente de desenvolvimento
\textit{Visual Studio}, tudo isso no sistema operacional \textit{Windows
10.}

Inicialmente, foi desenvolvido um software simples, com uma mistura
de janela-terminal. O usuário podia desenhar e simular, mas não tinha
muita liberdade para escolher e adicionar materiais. 
\begin{figure}[H]
\begin{centering}

\par\end{centering}
\caption{Versão 0.1, simples e utilizando a biblioteca \textit{SFML}}
\end{figure}

Conforme a evolução pedia, foi criada uma segunda janela, a qual replica
o desenho com as cores do material escolhido.

\begin{figure}[H]
\begin{centering}
\includegraphics[scale=0.6]{../imagens/v1\lyxdot 1}
\par\end{centering}
\caption{Versão 0.2, simples, mas preparando terreno para uma segunda janela
dos materiais.}
\end{figure}

Por fim, foi montada a versão final utilizando essa biblioteca. Foi
uma versão importantíssima para o aprendizado, pois o usuário não
desenhava diretamente no software, mas era enviado uma lista de propriedades
do desenho para o grid e, quando o desenho era atualizado, a biblioteca
utilizava os valores do grid. Isso permitiu juntar as duas janelas.

\begin{figure}[H]
\begin{centering}
\includegraphics[scale=0.8]{../imagens/v1\lyxdot 2}
\par\end{centering}
\caption{Versão 0.3, completa e complexa, mas muito lento.}
\end{figure}

Durante o desenvolvimento das versões anteriores, foi citado uma segunda
biblioteca gráfica chamada \textit{Qt}, mais rápida e completa que
a anterior. Então surgiu essa necessidade de mudança.

Como o software foi programado com orientação ao objeto, foi rápido
a migração, modificando, quase que somente, a classe da janela. Permitindo
criar o software na versão 1.0.
\begin{figure}[H]

\section{Diagrama de componentes}

O diagrama de componentes mostra as relações entre todos os componentes
do software.
\begin{figure}[H]
\begin{centering}
\includegraphics[width=0.9\textwidth]{\string"../imagens/Diagrama de componentes\string".png}
\par\end{centering}
\centering{}Fonte: Produzido pelo autor.\label{cap:Diagrama-de-componentes}
\end{figure}

Começando pela esquerda na Figura \ref{cap:Diagrama-de-componentes},
temos o executável e o main. Os outros dois componentes em azul criam
janelas para o usuário se comunicar com o software. O componente em
roxo, é a biblioteca de gráficos utilizados pelo software.

Os componentes em verde, são responsáveis por calcular as propriedades
dos materiais, por interpolação ou correlação.

Os componentes em laranja claro são responsáveis pela simulação da
difusão térmica.

quais equipamentos o software Simulador de Difusão Térmica utiliza.

o processador, memória RAM e memória de longo prazo, como um HD ou
